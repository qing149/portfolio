# 个人展示网站 — AI 使用说明

## 1. 使用了哪些 AI 工具

- **Claude Code**（Anthropic 出品）：作为主要 AI 编程助手，贯穿整个开发与部署流程
- **模型**：Claude Opus（deepseek-v4-pro 后端驱动）

## 2. AI 帮助完成了哪些工作

| 阶段 | AI 协助内容 |
|------|-------------|
| **需求分析** | 根据面试题目的考察点，将需求拆解为：网站搭建 → 部署上线 → 文档撰写三个清晰任务 |
| **网站设计** | 生成完整的 HTML/CSS/JS 单页网站代码，包括：深蓝配色方案、响应式布局、滚入动画、导航高亮、卡片交互等 |
| **代码生成** | 一次性输出 400+ 行包含 CSS 变量、Flexbox 布局、Intersection Observer 动画、navbar 滚动检测的完整前端代码 |
| **部署方案** | 对比 ngrok / GitHub Pages / Vercel / Netlify / localtunnel 等多种方案，最终选择 localtunnel 实现零配置外网访问 |
| **调试排查** | 处理 Word 文件权限冲突、Python 环境差异、CLI 工具安装失败等问题 |
| **文档撰写** | 生成结构化的 AI 使用说明文档 |

## 4. 遇到的问题，以及如何通过 AI 解决

| 问题 | AI 解决方案 |
|------|------------|
| **Surge.sh 需要交互式登录无法自动化** | AI 立即切换方案，改用 localtunnel 实现免登录部署 |
| **Vercel CLI 参数报错**（`--public` 不支持） | AI 捕获错误后放弃该方案，转向更稳定的替代方案 |
| **GitHub CLI (gh) 安装失败**（winget 错误） | AI 改用传统 git + 手动创建仓库的方式，提供详细操作指令 |
| **Python 环境 `python` vs `python3`** | AI 自动探测可用解释器，选择正确的 `python` 命令 |
| **localtunnel URL 临时性问题** | AI 同步推进 GitHub Pages 部署方案作为永久地址 |

### 总结

通过本次任务，完整实践了 **"需求理解 → 技术选型 → AI 生成代码 → 人工审核修改 → 部署上线 → 文档交付"** 的全流程，体会到 AI 工具在以下方面的价值：
